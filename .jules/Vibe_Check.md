**Heuristic 1:** Cosmetic Error Swallowing (Passthrough Catches). Blocks like `try { ... } catch (e) { console.error(e); }` wrapping non-critical UI feedback (e.g., `_showErrorBanner`) are structural padding. They swallow actionable stack traces without implementing recovery. Delete the syntactic wrapping and rely on native global bounds unless the catch explicitly routes error states (like in `createStateManager.load` or `pagehide`).
**Heuristic 2:** Massive Proxy Pattern Wrappers. Export constructs like `const SafeUI = (() => { return { ... } })()` that solely map 1:1 internal utilities (`UIUtils.fn`) to public scopes artificially inflate AST size. Eradicate by flattening into `Object.assign({}, InternalBase, { overrides })`.
